## コマンド備忘録
- `cat -n [ファイル]`: 行番号付きで中身を表示。
- `less [ファイル]`: スクロール表示。検索や移動が自由自在。
- `touch [ファイル]`: 空ファイルの作成。
- `rm -rf [対象]`: ディレクトリを警告なしで強制削除。
- `mv -i [元] [先]`: 上書き前に確認して移動・名前変更。
- `cp -r [元] [先]`: ディレクトリをコピー。
- `ln -s [実体] [リンク名]`: シンボリックリンク（ショートカット）を作成。
- `find [場所] -name '[名前]'`: 指定した名前でファイルを検索。

## mkdir と touch の違い
| コマンド | 作成するもの | イメージ |
| :--- | :--- | :--- |
| **mkdir** | ディレクトリ | 関連する書類をまとめるための **「空のバインダー（箱）」** を作る |
| **touch** | ファイル | 文字やコードを書き込むための **「白紙のレポート用紙」** を作る |
> **ポイント**: Java開発では `mkdir` でプロジェクトフォルダを作り、その中に `touch` でソースコード（.java）を作っていくのが基本の流れになります。

## テキスト vs バイナリ
- **テキストファイル**: 文字コードのみで構成される。Linuxの設定情報のほとんどはこれで管理されている。
- **バイナリファイル**: 画像、音声、実行ファイルなど。文字として読めない数値データを含むもの。

## リンクの仕組み
- **ハードリンク**: 1つのデータ実体に複数の「名前」をつける。本体を消してもデータは消えない。
- **シンボリックリンク**: データの「住所（パス）」が書かれたメモ。本体を消すとリンク切れになる。開発現場ではこちらが主流。

## lessコマンドの操作表
| キー | 操作 |
| :--- | :--- |
| **Space / f** | 一画面下にスクロール |
| **b** | 一画面上にスクロール |
| **j / k** | 一行下 / 一行上に移動 |
| **/<文字列>** | 下方向に検索 |
| **?<文字列>** | 上方向に検索 |
| **n / N** | 次の検索結果 / 前の検索結果に移動 |
| **q** | 終了 |
