# Day4 学習メモ：標準入出力とパーミッション

## 1. 標準入出力とリダイレクト
Linuxのコマンドには3つのデータの通り道（土管）があり、これをつなぎ変えることができます。

- **標準入力(0)**: キーボード等からの入力。 `<` でファイルに切替可能。
- **標準出力(1)**: 実行結果の表示。 `>` や `>>` でファイルに切替可能。
- **標準エラー出力(2)**: エラー時の表示。 `2>` でファイルや `/dev/null`（ゴミ箱）へ送れる。
- **パイプライン(`|`)**: 左のコマンドの出力を、右のコマンドの入力へ直接渡す機能。

### 演習で使ったコマンド
- `ls /bin | wc -l`: ディレクトリ内のファイル数をカウント。
- `ls /etc | grep host | wc -l`: `/etc` 内で `host` という文字を含むファイル数をカウント。
- `cat < /etc/hosts > output.txt`: ファイルの中身を読み込んで、別のファイルへ出力（コピー）。

## 2. パーミッション（権限）の基本
ファイルやディレクトリに対して「誰が何をしていいか」を管理します。

### 権限の3要素
- **r (read)**: 読み取り権限（中身を見る）
- **w (write)**: 書き込み権限（編集・削除する）
- **x (execute)**: 実行権限（プログラムとして動かす / ディレクトリに入る）

### 対象の3グループ
`ls -l` で表示した際、左から順に以下の権限を示しています。
1. **所有者 (User)**: 自分
2. **グループ (Group)**: 同じグループの仲間
3. **その他 (Others)**: それ以外の全員

### よく使うコマンド
- `chmod [権限] [ファイル名]`: 権限を変更する（例: `chmod 755 script.sh`）
- `chown [ユーザー名] [ファイル名]`: 所有者を変更する
